# Photo Archive Evaluator — 명세서 (SPEC)

> 본 문서는 프로젝트의 단일 사실 출처(Single Source of Truth)이다.
> 변경 시 반드시 PR로 검토하고, 머지되면 위키에 자동 반영된다.

---

## 1. 목적

NAS에 저장된 사진을 자동으로 인식·평가하여 다음 의사결정을 지원한다.

- 포트폴리오 구성에 적합한 사진 선별
- 주제·시리즈별 자동 정리
- 공모전 출품 후보 선별 및 카테고리별 적합도 평가

## 2. 용어

| 용어 | 의미 |
|---|---|
| 기본 평가 | local AI 모델 또는 외부 API의 1차 평가(미학 점수, 캡션, 태그, 임베딩) |
| 고급 평가 | 사용자 선택 사진에 대해 상위 모델로 수행하는 상세 리뷰 |
| 임베딩 | 사진의 의미 표현 벡터. 시맨틱 검색·클러스터링용 |
| pHash | 지각적 해시. 동일 사진의 리사이즈/포맷 변환 식별 |
| 표시 임계값 | 기본 사진 목록에 노출할 최저 점수(기본 4점) |
| 평가 결과 | 모델 산출물 전체. 점수와 무관하게 전부 저장 |

## 3. 사용 시나리오

1. 사용자는 웹 UI에서 NAS 접속 정보·대상 폴더·실행 일정을 설정한다.
2. 프로그램은 지정된 시간대에 NAS를 탐색하고 신규/변경 사진을 식별한다.
3. 식별된 사진은 평가 큐로 들어가 기본 평가가 수행된다.
4. 모든 평가 결과는 저장되고, 표시 임계값 이상이 1차 목록에 노출된다.
5. 사용자는 목록에서 검색·분류·필터링한다.
6. 사용자는 특정 사진을 선택해 고급 평가를 요청하고 결과를 확인·태그한다.
7. 사용자는 결과를 다운로드(원본/리스트)하거나 export(엑셀/DB) 한다.

## 4. 기능 요구사항

### 4.1 프로그램 동작
- PC가 켜져 있고 사용자가 지정한 요일·시간대일 때 자동 실행
- 시간대 외 도래 시 진행 중 작업은 다음 회차로 이월
- 절전/슬립 진입 시 작업 일시정지, 복귀 시 자동 재개
- 일정 누락 catch-up 정책: **다음 정시까지 대기**(기본). 사용자가 "즉시 보충" 옵션 선택 가능
- 1인 사용 가정. LAN 내 다른 단말의 접근 차단을 위해 웹 UI 비밀번호 보호 필수

### 4.2 NAS / 사진 탐색
- Synology DSM Web API (FileStation) 사용
- mDNS/SSDP 기반 NAS 자동 검색 또는 사용자 IP 입력
- 다수 대상 폴더 지정. 하위 폴더 재귀 탐색
- 자격증명: ID/PW 또는 API 토큰. 2FA 지원. OS 키체인/암호화 저장
- 동시성 상한: `max_concurrent_workers`(기본 4), `max_requests_per_second`(기본 2) 설정값 준수
- 1차 지원 포맷: **JPEG (.jpg/.jpeg)**
- 그 외 포맷(HEIC/RAW/PNG/TIFF)은 카운트만 기록하고 평가 보류
- 제외 규칙: 특정 폴더·파일·패턴 제외(예: `**/_archive/**`)

### 4.3 동일 사진 식별 / 변경 감지
- 1차 키: `(size, mtime)` 캐시 — 동일하면 변경 없음으로 간주
- 2차 키: SHA-256 — 1차 캐시가 깨지거나 의심 시 검증
- 3차 키: pHash — 리사이즈/이동/이름변경 추적
- NAS에서 삭제·이동된 사진은 즉시 삭제하지 않고 `missing` 상태로 표시. pHash 매칭으로 자동 복구

### 4.4 기본 사진 인식·평가
- 어댑터 인터페이스로 local 모델 또는 외부 API 선택
- 산출물:
  - **EXIF**: 촬영일시, 카메라/렌즈, 노출/조리개/ISO/초점거리, GPS
  - **AI 결과**: 미학 점수(1–5), 캡션, 태그(다중), 구도/색감/주제 분류, 임베딩 벡터
  - **운영 정보**: 사용 모델 ID/버전, 평가 시각, 신뢰도
- **모든 평가 결과를 저장**한다(점수 무관). 표시는 임계값(기본 4점) 이상.
- 임계값은 사용자가 조정 가능하며, 변경 시 목록은 즉시 재필터링된다.
- 신뢰도 낮은 결과는 별도 큐로 분리하여 사용자 검토 유도

### 4.5 자동 분류
- **포트폴리오 그룹**: AI 추천 + 사용자 확정. 한 사진이 여러 그룹에 속할 수 있음
- **공모전 카테고리** (인물/풍경/스트리트/흑백 등): CLIP 기반 zero-shot 다중 라벨 분류, 적합도 점수 동반
- **시리즈 클러스터링**: 임베딩 기반 자동 그룹화. 사용자 검토·수정 가능

### 4.6 고급 사진 리뷰·평가
- 사용자가 1장 또는 다수 선택해 고급 모델로 평가
- 호출 직전 **추정 비용 미리보기** + 사용자 확인 다이얼로그
- 결과는 별도 테이블에 기록(같은 사진에 다중 모델·다중 시도 보존)
- 사용자가 결과 검토 후 사진에 태그·코멘트 추가 가능

### 4.7 정보 저장 / 백업
- 메인 DB: PC 로컬 SQLite
- NAS 백업: 주기 백업(기본 1일 1회) — DB 덤프 + 임베딩 / 썸네일 폴더
- 보관 세대: 최근 N일치(기본 14일) + 월별 1개 6개월
- UI에서 임의 시점 백업 선택해 복원
- Export: SQLite 덤프, Excel(xlsx)

### 4.8 검색 / 분류
- **키워드 검색**: 캡션·태그·EXIF·파일명 통합 인덱싱
- **시맨틱 검색**:
  - 텍스트 → 사진 (예: "석양에 실루엣")
  - 사진 → 비슷한 사진
- 점수·카테고리·기간·카메라·GPS 영역 필터
- 검색 결과를 리스트로 저장, 리스트 단위 다운로드(ZIP) 또는 export

### 4.9 사용자 UI
- 웹 인터페이스(브라우저)
- 비밀번호 인증
- 페이지: Dashboard / Library / Search / Portfolio / Advanced / Settings / Logs
- 썸네일 생성·캐싱(첫 평가 시 생성, 로컬에 LRU 캐시)
- AI 점수와 별도로 **사용자 점수 오버라이드** 입력 가능
- 정렬·필터의 정렬 기준에 `ai_score` / `user_score` / `final_score`(우선순위 사용자 점수) 선택
- Dry-run 모드: 실제 평가 없이 "몇 장이 평가될지"만 미리보기

### 4.10 설정
- NAS 접속 정보 / 자동 검색
- 대상 폴더 다수 지정, 제외 규칙
- 실행 요일·시간대, 동시성 상한
- AI 모델 선택(local/외부) 및 모델별 옵션
- 외부 전송 동의, EXIF / GPS 제거 옵션, 인물 자동 차단 옵션
- API 키 보관(키체인)
- 비용 한도(월/회당), 비용 미리보기 토글
- 표시 임계값
- 백업 주기·세대·복원

## 5. 비기능 요구사항

### 5.1 성능 / 동시성
- DS218+ 부하 보호: 동시 워커 N(기본 4), 초당 요청 R(기본 2) 상한
- 우선순위: 신규 > 변경 > 재평가
- 큐 백프레셔: 큐 길이 상한 시 탐색 일시정지

### 5.2 회복성 / 재개
- 작업 단위 상태(`pending` / `in_progress` / `done` / `failed`) 트랜잭션
- 비정상 종료 시 시작 시 `in_progress` → `pending` 복구
- 실패 N회(기본 3) 초과 시 `failed`로 격리, 알림
- NAS 단절: 지수 백오프 재시도. 큐 일시정지

### 5.3 보안
- 웹 UI 비밀번호 보호(bcrypt + 세션 쿠키, HttpOnly·SameSite=Lax)
- 모든 자격증명(NAS, 외부 API)은 OS 키체인(Windows DPAPI / Linux libsecret) 또는 fallback AES 키파일(권한 0600)
- 외부 전송 시 GPS·민감 EXIF 제거 옵션
- 인물 자동 감지 시 외부 전송 차단 옵션

### 5.4 로깅 / 모니터링
- 구조화 로그(JSON): app.log
- 작업 로그(시각, 대상, 결과, 비용)
- 에러 로그(스택, 컨텍스트)
- 사용자 가시 화면: 진행률, 큐 길이, 마지막 스캔 시각, 오늘의 처리량, 실패 N건
- 일/주 단위 처리량·비용 그래프
- 로그 다운로드
- `/healthz`, `/metrics` 엔드포인트

### 5.5 개인정보
- GPS·인물 정보 외부 전송 제어
- 모든 외부 전송은 사용자 동의 설정 ON일 때만

### 5.6 다국어
- UI 한국어 기본, 영어 옵션
- AI 캡션·태그는 모델 출력 그대로 저장(언어 자동 감지)

## 6. AI 모델

### 6.1 1차 추천 모델 (local)

| 용도 | 모델 | 비고 |
|---|---|---|
| 미학 평가 | Aesthetic Predictor V2.5 (SigLIP 기반) | ~1GB, 1–5점 매핑 |
| 임베딩 | CLIP ViT-L/14 또는 SigLIP | 검색·클러스터링용 |
| 캡션 | BLIP-2 또는 Qwen2-VL-2B-AWQ | RTX 4050 6GB 적합 |
| Zero-shot 카테고리 | CLIP 텍스트 매칭 | 공모전 카테고리 |

### 6.2 고급 평가 (외부 API)
- Claude Sonnet 4.6 / Opus 4.7 (vision)
- GPT-4o
- Gemini 2.0
- 어댑터로 추가 가능

### 6.3 어댑터 인터페이스 (4종)
- `ScoreModel.score(image) -> {score, confidence}`
- `CaptionModel.caption(image) -> {caption, lang, tags[]}`
- `EmbeddingModel.embed(image) -> {vector, model_id, dim}`
- `AdvancedReviewModel.review(image, prompt) -> {response, cost, model_id}`

### 6.4 추론 백엔드
- ONNX Runtime + DirectML(Windows GPU) / CUDA
- llama.cpp / ollama 옵션(VLM 캡션용)

### 6.5 임베딩 버전 관리
- 임베딩 레코드에 `model_id`, `model_version` 저장
- 모델 교체 시 사용자에게 "재임베딩 필요" 경고
- 백그라운드 점진적 재임베딩 가능

## 7. 배포 환경

- Windows PC: Intel Core Ultra 7 155H, RTX 4050 6GB
- NAS: Synology DS218+ (RAM 2GB)
- OS: Windows 11(주), Linux(개발/테스트)

## 8. 1차 범위 제외 (향후 확장)

- RAW/HEIC/TIFF/PNG 평가
- 다중 사용자 / 권한 분리
- 모바일 전용 UI
- 사진 자동 보정/편집
- 클라우드 외부 저장(S3 등) 백업
- AI 모델 fine-tune

## 9. 성공 기준

- 1만 장 라이브러리 1차 스캔 24시간 이내 완료
- 미학 점수의 사용자 동의율(±1점) ≥ 70%
- 시맨틱 검색 top-10 적중률(주관 평가) ≥ 80%
- NAS 평균 부하 50% 이하 유지
- 7일 연속 무인 운영 시 장애 없음
