| 통계 | 값 |
|---|---|
| 최저 | 4.344 |

명목 1–10 스케일이지만 **실제 분포는 4.3–7.0에 집중**. 이는 사진 라이브러리 일반에서 관찰되는 특성이다.

## 정규화 공식 결정

**변경 전(초안)**: `score = raw / 2` → 모든 값이 2.2–3.5 범위 → 4점 이상 0%

**변경 후(채택)**: `score = clamp(raw − 2, 1, 5)`

대응 의미:
- raw 5 → 3점 (decent, 라이브러리 가치)
- raw 6 → 4점 (strong, 공모전급)
- raw 7 → 5점 (excellent, 포트폴리오급)

raw_score는 DB에 항상 보존하므로 사용자 라이브러리에 맞춘 percentile 기반 재캘리브레이션이 추후 가능하다.

코드: [backend/app/ai/local/aesthetic.py](https://github.com/hallower/PhotoArchiveEvaluator/blob/main/backend/app/ai/local/aesthetic.py)

## 후속 작업

1. **사용자 검증**: 동일 113장 중 무작위 30장에 대해 사용자가 1–5점 직접 매기고 동의율 확인 → SPEC 성공 기준 ±1점 70% 충족 여부 결정
2. **임계값 조정 검토**: 43%가 너무 관대하면 4.0 → 4.3 (raw 6.3) 정도로 상향
3. **Phase 1 백엔드 골격 시작**: FastAPI + SQLite + DSM FileStation 클라이언트 + 비밀번호 인증
4. **Linux/CPU fallback 검증**: 추후 추가 환경 확인

## 알려진 제약

- HuggingFace 다운로드는 비인증 요청 사용. 운영 시 `HF_TOKEN` 설정 권장
- Windows에서 HuggingFace 심링크 미지원 경고 — 캐시 디스크 사용량 약간 증가 (무시 가능)
- 콘솔 한글 출력 깨짐(cmd cp949) — 본 PoC 스크립트는 영문 출력으로 변경. 본 운영 백엔드는 UTF-8 웹 UI 사용 예정이라 무관
