# MVP 단계 정의

본 프로젝트는 **4단계로 점진적**으로 확장한다. 각 단계는 독립 릴리즈 가능하며,
다음 단계 진입 전 기능 검증·실사용 후 결정한다.

검증되지 않은 기능을 한 번에 구현하지 않는 것이 목적이다 — 매 단계는 사용자가
손에 쥐고 시험할 수 있는 동작 단위로 정의된다.

---

## Phase 1 — MVP (최소 동작)

**목표**: NAS의 JPG 사진을 자동으로 점수화하고 기본 목록·검색이 가능

### 포함
- 설정: NAS IP/계정, 단일 대상 폴더, 1회 수동 스캔
- DSM FileStation API 접속, JPG 재귀 탐색
- 동일 사진 식별: `size+mtime` 1차 캐시 + SHA-256
- 기본 평가: **미학 점수만** (Aesthetic Predictor V2.5)
- 모든 점수 저장. 표시 임계값(4점) 필터
- 메인 DB: SQLite (PC 로컬)
- 웹 UI: 라이브러리 목록, 점수·날짜 정렬, 썸네일 생성·캐싱
- 비밀번호 인증
- 로깅: 콘솔 + 파일

### 제외
- 캡션/태그/임베딩
- 공모전 카테고리/포트폴리오
- 고급 평가
- NAS 백업
- 자동 일정
- pHash

### 검증 기준
- 1,000장 스캔 1시간 이내
- NAS 평균 부하 50% 이하
- 점수 분포 시각 검증(히스토그램, 사람 검토 30장 표본)

---

## Phase 2 — 의미 검색 / 분류

**목표**: 검색·분류가 의미 있게 동작

### 추가
- CLIP 임베딩 + BLIP-2 캡션
- 태그 다중 라벨
- 키워드 검색(캡션·태그·EXIF)
- 시맨틱 검색(텍스트 → 사진)
- pHash 추가 → 이동·이름변경 추적
- 공모전 카테고리 zero-shot 분류
- 사용자 점수 오버라이드
- 다수 대상 폴더, 자동 일정(요일·시간대)
- 동시성 상한 설정
- 재개 로직(상태 트랜잭션)

### 검증
- 시맨틱 검색 주관 적중률 ≥ 80% (질의 20개 표본)
- 자동 일정 7일 무인 동작
- PC 절전·복귀 시 작업 정상 재개

---

## Phase 3 — 고급 평가 / 백업 / 운영성

**목표**: 외부 API 고급 평가 및 안정 운영

### 추가
- 외부 API 어댑터(Claude / GPT / Gemini)
- 고급 평가 UI: 비용 미리보기, 결과 별도 저장, 다중 시도 보존
- 비용 한도/추적, 월 한도 도달 시 차단
- 외부 전송 동의 / GPS·EXIF 제거 / 인물 차단
- NAS 자동 백업(DB 덤프 + 임베딩 / 썸네일), 복원 UI
- 사진 → 비슷한 사진 검색
- 포트폴리오 그룹(수동 + AI 추천)
- 시리즈 자동 클러스터링
- Export(엑셀)
- Health check 대시보드, Dry-run 모드

### 검증
- 비용 한도 초과 차단 동작 시뮬레이션
- 백업 → 복원 라운드트립 검증(동일 해시 재현)
- 외부 전송 차단 옵션이 켜진 상태에서 실제로 송신되지 않는지 네트워크 캡처로 확인

---

## Phase 4 — 확장

- RAW / HEIC / PNG / TIFF 지원
- 임베딩/모델 교체 시 점진적 재임베딩 잡
- 모바일 친화 UI
- AI 점수 사용자 보정 데이터로 모델 fine-tune 실험
- Linux 정식 배포 패키지(deb / rpm 또는 Docker compose)

---

## 단계 간 마이그레이션 정책

- 모든 스키마 변경은 alembic 마이그레이션으로 관리
- 단계 진입 시 기존 데이터를 보존한다(대규모 재처리는 백그라운드 잡)
- 사용자는 항상 한 단계 이전 상태로 롤백 가능해야 한다(백업 보존)
